### AI 转译链路（核心合规引擎）

```
用户输入（大白话）
     │
     ▼
┌──────────────────────┐
│ 第一关：词库熔断        │     命中 → 🛑 拦截，不提交
│ mud_sensitive_words    │     未命中 ↓
└────────────┬─────────┘
             ▼
┌──────────────────────┐
│ 第二关：AI 转译引擎     │
│ ① 提取情绪+事件要素    │
│ ② 强制代换（地名→架空） │
│ ③ 套用 MUD 黑话模板    │
│ ④ 第三人称客观叙述     │
└────────────┬─────────┘
             ▼
┌──────────────────────┐
│ 第三关：输出合规校验     │     含煽动/号召 → 🛑 替换为安全文案
│ 检测祈使句/号召性话语    │     通过 ↓
└────────────┬─────────┘
             ▼
       传闻展示在信息流
```

### 异常链路

```
江湖传闻:
  输入含敏感词 → 熔断拦截 → 返回错误提示「此言触犯江湖禁忌」
  AI 转译超时 → 降级展示原文（不含 MUD 化） → 后台异步重试
  AI 输出含号召语 → 替换为安全文案 → 标记供人工复审

恩怨台:
  qi=0 → 拒绝进入 → 提示「内力枯竭」
  qi 不足支付战斗费用 → 拒绝 → 提示「真气不足」
  NPC 生成失败 → 降级使用预置 NPC 模板

悬赏布告:
  发布无位置信息 → 降级为纯文字悬赏（无导航）
  接取已被他人接取 → 409 乐观锁
  完成时非接取人 → 400 拒绝
  超时未完成 → 系统自动 expired

角色创建:
  已有档案 → 409「已有江湖档案」
  无效职业代码 → 400「无效职业」
```

| # | 类别 | 条件 | 预期行为 | 优先级 |

| B1 | 合规 | 用户绕过前端直接调 API 提交敏感内容 | 后端同样有词库校验（双重防线） | P0 |

## Spike 建议

1. **AI 转译引擎 Spike** (限时 4h)
   - 目标: 确认 Prompt 模板可行性，评估转译质量
   - 产出: 3 套 Prompt + 10 条样本输入输出 + 质量评分
   - 验证: 地名代换成功率 >95%，平台名代换 100%

2. **合规管线性能 Spike** (限时 2h)
   - 目标: 验证词库熔断 + AI 双层管线的端到端延迟
   - 产出: 性能基准数据 (p50/p95/p99)
   - 验证: 词库匹配 <10ms，总流程 <5s

3. **超代订单 API Spike** (限时 2h)
   - 目标: 确认悬赏→订单→返利的数据流是否可行
   - 产出: API 契约文档 + 数据映射表
   - 判定: 若 API 不可用，悬赏 V1.0 降级为纯社区情报（无订单绑定）
