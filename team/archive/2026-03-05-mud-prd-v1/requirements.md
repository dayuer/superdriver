# 需求：超凡驿站 MUD 游戏化社区 PRD (V1.0)

> 🎯 PM Phase 1 · 2026-03-05  
> 复杂度：**L 级** — 跨业务域 × AI 内容合规 × 游戏化经济系统 × 多职业体系

---

## 背景

运力群体（代驾/网约车/外卖骑手）在工作中承受高压，缺乏合规的情绪出口。传统论坛社区模式面临内容合规风险高、用户粘性低、难以反哺业务三大问题。

本需求通过 MUD 文字冒险的形式，将现实痛点包装为武侠江湖叙事，借助 AI 文本重构实现"合规化表达"，同时通过游戏化的悬赏机制驱动跨职业流量裂变。

### 业务价值

1. **情绪安全阀** — 用 AI 文本重构把高风险抱怨转化为合规的"江湖纪事"
2. **圈层壁垒** — 用黑话体系建立身份认同，提升留存率
3. **业务引擎** — 通过悬赏机制驱动跨职业订单裂变

---

## 用户故事

### US-1: 江湖传闻（UGC 吐槽与共鸣）

**作为** 一名运力从业者，**我希望** 在输入大白话抱怨后系统自动生成武侠风的"江湖传闻"，**以便** 我能合规地表达工作中的情绪，并与同行产生共鸣。

#### 验收标准

- [ ] **AC-1.1**: Given 玩家已登录且有 MUD 档案, When 在输入框输入一段大白话抱怨并提交, Then 系统调用 AI 接口生成一条第三人称客观叙述的江湖传闻并展示在信息流中
- [ ] **AC-1.2**: Given 传闻流中有一条传闻, When 点击 [递上一碗烈酒], Then 该传闻的同情计数 +1 并显示动效反馈
- [ ] **AC-1.3**: Given 传闻流中有一条传闻, When 点击 [拔刀怒骂], Then 该传闻热度值 +1 并显示视觉反馈
- [ ] **AC-1.4**: Given 传闻流中有一条传闻, When 点击 [围炉夜话] 并输入评论, Then 评论内容同样经过 AI 黑话处理后展示
- [ ] **AC-1.5**: Given AI 转译引擎接收到含真实地名的文本, When 执行转译, Then 所有现实地名被强制代换为架空地名（如"北京朝阳区"→"中州皇城东角"）
- [ ] **AC-1.6**: Given AI 转译引擎接收到含平台真名的文本, When 执行转译, Then 所有平台名被代换为门派名（如"滴滴"→"苍水阁"）

#### 优先级: P0

---

### US-2: 恩怨台（合规的宣泄出口）

**作为** 一名遭遇不公（恶意投诉/无理罚款）的司机，**我希望** 能在恩怨台上消耗真气生成一个 AI 假人并用武功招式虐打它，**以便** 我能安全地宣泄负面情绪并获得积分奖励。

#### 验收标准

- [ ] **AC-2.1**: Given 玩家有档案且 qi > 0, When 进入恩怨台, Then 消耗指定数量 qi 并生成一个基于事件类型的 AI NPC（含名称/类型/对话/弱点）
- [ ] **AC-2.2**: Given NPC 已生成, When 玩家点击文字技能（如 [拒载飞踹]、[绕路迷阵]）, Then 系统输出一段爽快的武侠战斗文本
- [ ] **AC-2.3**: Given 战斗进行中, When 战斗结束, Then 玩家永远胜利，获得"碎银"积分和爽文结算画面
- [ ] **AC-2.4**: Given 玩家 qi = 0, When 尝试进入恩怨台, Then 拒绝进入并提示"内力枯竭，请先完成悬赏恢复真气"

#### 优先级: P0

---

### US-3: 悬赏布告（跨界引流与业务裂变）

**作为** 一名在某地点发现商机的骑手/司机，**我希望** 能在客栈发布情报悬赏，让其他职业的玩家前往该地接单，并在他们成功时获得返利，**以便** 不同职业之间能互相引流产生真实业务价值。

#### 验收标准

- [ ] **AC-3.1**: Given 玩家有档案, When 发布一条情报悬赏（含位置描述和职业标签）, Then 悬赏以 MUD 黑话风格展示在悬赏榜上
- [ ] **AC-3.2**: Given 悬赏榜上有 open 状态的悬赏, When 目标职业玩家点击接取, Then 悬赏状态 → accepted，接取者可见位置引导
- [ ] **AC-3.3**: Given 接取者已到达目标位置, When 通过超代系统成功完成订单, Then 悬赏状态 → completed，发布者自动获得积分/现金返利
- [ ] **AC-3.4**: Given 悬赏超过 24h 未被接取, When 系统定时检查, Then 悬赏状态 → expired
- [ ] **AC-3.5**: Given 玩家浏览悬赏榜, When 按职业标签过滤, Then 仅展示对应职业可接取的悬赏

#### 优先级: P1

---

### US-4: 世界观与角色系统

**作为** 一名新用户，**我希望** 能选择一个与我真实职业对应的江湖职业并获得专属称号和技能，**以便** 我在社区中拥有明确的角色身份认同。

#### 验收标准

- [ ] **AC-4.1**: Given 新用户进入 MUD 模块, When 浏览职业引导页, Then 展示三大职业（夜行镖师/铁骑游侠/神行游侠）含描述、初始属性和专属技能
- [ ] **AC-4.2**: Given 用户选择职业并起名, When 确认创建, Then 创建 MUD 档案，初始属性: qi=100, silver=0, rank="初入江湖"
- [ ] **AC-4.3**: Given 用户已有 MUD 档案, When 查看个人主页（藏经阁）, Then 展示职业、真气值、碎银数、当前称号、战绩、装备

#### 优先级: P0（已部分实现 ✅）

### US-5: AI NPC 阵营体系

**作为** 系统设计者，**我希望** AI 能控制一系列 NPC 阵营并在传闻和恩怨台中自然出现，**以便** 玩家的吐槽指向虚拟 NPC 而非真实个体/企业，确保合规。

#### 验收标准

- [ ] **AC-5.1**: Given 系统初始化, When 创建世界, Then 预置三类 NPC 阵营：大宗门（平台 NPC）、散人（乘客/食客 NPC）、市井阻碍（现实痛点 NPC）
- [ ] **AC-5.2**: Given 玩家发帖提及现实平台/角色, When AI 转译, Then 自动替换为对应 NPC 阵营的角色名
- [ ] **AC-5.3**: Given NPC 阵营数据配置, When 管理员在 Boss 后台修改, Then mud_factions 表同步更新，前端下次加载生效

#### 优先级: P1（管理端已部分实现 ✅）

---

### US-6: 合规与风控红线

**作为** 平台运营方，**我希望** 所有用户输入都经过多层合规过滤，**以便** 社区内容不会引发法律/舆情风险。

#### 验收标准

- [ ] **AC-6.1**: Given 用户输入包含敏感词（真实平台全名/管理层真名/政治/黄赌毒）, When 提交, Then 输入端直接拦截，不提交至 AI 处理
- [ ] **AC-6.2**: Given AI 转译完成, When 输出包含煽动罢工/聚集/暴力对抗的祈使句或号召性话语, Then 系统过滤该输出并返回安全替代文案
- [ ] **AC-6.3**: Given 用户注册或进入客栈, When 页面加载, Then 常驻显示免责声明："本驿站内所有传闻皆为赛博江湖虚构故事…如有雷同，纯属巧合"
- [ ] **AC-6.4**: Given 敏感词库需迭代, When 管理员在 Boss 后台更新, Then mud_sensitive_words 表同步更新，即时生效无需重启

#### 优先级: P0

---

### US-7: UI 风格与信息层级

**作为** 用户，**我希望** 界面采用复古文字风暗黑模式，信息层级清晰，**以便** 沉浸体验。

#### 验收标准

- [ ] **AC-7.1**: Given 进入 MUD 主页（客栈大堂）, When 页面加载, Then 顶部显示个人状态栏（职业/真气/碎银/称号），主体为传闻流，底部悬浮输入框
- [ ] **AC-7.2**: Given 主页, When 打开侧栏/底部导航, Then 可进入悬赏榜、恩怨台、藏经阁三个子模块
- [ ] **AC-7.3**: Given 任何 MUD 页面, When 渲染文本, Then 使用复古终端绿字或羊皮纸黑字风格（可切换）
- [ ] **AC-7.4**: Given 用户进行任何操作, When 操作完成, Then 有符合 MUD 世界观的动效和音效反馈

#### 优先级: P2

## 技术风险标注（L 级）

| # | 风险 | 影响 | 建议 |
|---|------|------|------|
| R1 | AI 转译质量不可控 | 输出可能不符合合规要求或风格不一致 | Spike: 供给 prompt 模板 + 输出质量评估 |
| R2 | 实时内容合规检测延迟 | 高并发下 AI 处理可能超时 | Spike: 前置词库拦截 vs AI 后置审查的性能权衡 |
| R3 | 悬赏与真实订单对接 | 需要与超代系统的订单 API 打通 | Spike: 确认超代系统 API 可用性和数据格式 |
| R4 | 地名代换完整性 | 中国城市/区划名称库庞大 | 可用 AI 能力 + 高频地名字典组合 |
| R5 | 跨职业返利结算 | 涉及真实金钱流转，需审计 | 初期仅做积分返利，不涉及现金 |
