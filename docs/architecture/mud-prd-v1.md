## 1. 产品概述

### 1.1 定位

一款内嵌于"超代"应用中的极简 MUD 文字社区。以架空武侠/赛博江湖为背景，为代驾、网约车、外卖骑手提供一个兼具匿名吐槽、情绪发泄与线索交易的虚拟客栈。

### 1.2 核心目标

| 目标 | 描述 | 衡量指标 |
|------|------|----------|
| **情绪出口** | 通过 AI 文本重构，将高风险行业抱怨转化为合规的"江湖纪事" | 社区日活、传闻发布量、合规拦截率 |
| **圈层固化** | 用特有黑话体系建立身份认同，形成社区壁垒 | 7 日留存率、互动率（烈酒/怒骂/围炉） |
| **业务赋能** | 通过悬赏机制驱动跨职业流量交叉与订单裂变 | 悬赏完成率、跨职业转化率 |

### 1.3 目标用户

运力从业者群体 — 代驾司机、网约车司机、外卖骑手。

## 3. 核心玩法设计

### 3.1 江湖传闻（UGC 吐槽与共鸣）— P0

最高频功能，替代传统论坛发帖。

#### 机制

```
用户大白话 → [词库熔断] → [AI 转译] → [合规校验] → 江湖传闻
```

1. 玩家在输入框写大白话抱怨
2. **第一关 — 词库熔断**: 输入端匹配 `mud_sensitive_words`，命中即拦截
3. **第二关 — AI 转译**: 提取情绪+事件要素 → 强制代换地名/平台名 → 套用黑话模板 → 第三人称客观叙述
4. **第三关 — 输出合规**: 检测祈使句/号召性话语，命中则替换为安全文案

#### 互动按钮

| 按钮 | 语义 | 数据操作 |
|------|------|----------|
| [递上一碗烈酒] | 同情/安慰 | `community_interactions` type=sympathy |
| [拔刀怒骂] | 同仇敌忾 | `community_interactions` type=rage，增加热度值 |
| [围炉夜话] | 评论回复 | `community_interactions` type=comment + AI 黑话化 |

#### 关键约束

- 所有互动评论同样经 AI 转译管线处理
- 前端防抖 + 后端幂等键（30s 内相同内容去重）
- 单用户每分钟最多 3 条传闻

---

### 3.2 恩怨台（合规的宣泄出口）— P0

针对司机被恶意投诉/无理罚款后无处发泄的痛点。

#### 机制

1. 玩家进入恩怨台，消耗 qi（真气）
2. 系统基于事件类型生成 AI NPC（含名称/类型/对话/弱点）
3. 玩家点击文字职业技能进行战斗
4. **结果永远是玩家胜利**，输出爽快击杀文本 + 掉落碎银

#### 规则

| 规则 | 说明 |
|------|------|
| qi 熔断 | qi = 0 时拒绝进入恩怨台 |
| 日次限制 | 每用户每日最多 N 次（可配置） |
| NPC 降级 | AI 生成失败时使用预置 NPC 模板 |
| 必胜保证 | 战斗结果固定为玩家胜利 |

---

### 3.3 悬赏布告（跨界引流与业务裂变）— P1

将游戏道具与超代的真实拉新/接单业务绑定。

#### 状态机

```
[发布] → open → [接取] → accepted → [完成] → completed
                                   → [超时48h] → expired
         → [超时24h] → expired
```

#### 跨职业引流场景

> 神行游侠（外卖员）在夜市送餐，发现某酒楼生意火爆。在客栈发布：
> **[探子密报]** 东街醉仙楼今夜豪客云集，急需夜行镖师前往护阵。
>
> 夜行镖师接取后前往该地，通过超代成功接单 → 发布者获得积分返利。

#### 关键约束

| 约束 | 说明 |
|------|------|
| 并发接取 | 数据库 CAS/乐观锁，后到者 409 |
| 职业过滤 | 悬赏按 `target_profession` 过滤展示 |
| 同时接取上限 | 单用户最多 3 个同时进行 |
| V1.0 返利 | 仅积分返利，不涉及现金（合规考量） |

## 5. 合规与风控方案（红线防御）

> ⚠️ 本模块是项目生存的底线，必须严格执行。

### 5.1 三层合规防线

| 层级 | 机制 | 时机 | 延迟目标 |
|------|------|------|---------|
| **L1 — 词库熔断** | `mud_sensitive_words` 精确/模糊匹配 | 输入端，提交 AI 之前 | <10ms |
| **L2 — AI 强制代换** | Prompt 模板中硬编码代换规则 | AI 转译过程中 | — |
| **L3 — 输出合规校验** | 检测祈使句/号召性话语 | AI 输出后、展示前 | — |

### 5.2 词库熔断规则

| 类别 | 示例 | 处理 |
|------|------|------|
| 平台全名 | 滴滴出行、美团外卖、高德打车 | 拦截 |
| 管理层真名 | 各平台 CEO/VP 真名 | 拦截 |
| 政治敏感 | 由安全团队维护 | 拦截 |
| 黄赌毒 | 标准违规关键词 | 拦截 |

### 5.3 强制代换映射

| 现实 | 架空 | 数据来源 |
|------|------|---------|
| 北京朝阳区 | 中州皇城东角 | AI + 地名字典 |
| 运管 | 巡街官差 | `mud_factions` real_mapping |
| 滴滴 | 苍水阁 | `mud_factions` real_mapping |
| 乘客 | 散人 | AI Prompt 模板 |

### 5.4 客观陈述原则

AI 生成内容必须满足：
- ✅ 第三人称客观叙述
- ❌ 不含煽动罢工/聚集/暴力对抗的祈使句
- ❌ 不含号召性话语（"大家一起"/"让我们"）

### 5.5 免责声明

> "本驿站内所有传闻皆为赛博江湖虚构故事，各路游侠切莫对号入座，如有雷同，纯属巧合。"

- 注册时弹窗确认
- 客栈大堂底部常驻小字
